# Demo Readiness Plan

## เป้าหมายหลัก

ทำให้ระบบ AI Smart Home เป็นผู้ช่วยภาษาไทยที่คุยได้จริง เดโมได้ลื่น และยังคุมบ้าน/อ่านข้อมูลสดได้อย่างน่าเชื่อถือ โดยเน้นเสถียรภาพ ความเร็ว และประสบการณ์เหมือนคุยกับผู้ช่วยจริงมากขึ้น

## หลักการทำงาน

1. ทำทีละชิ้นเล็ก และทดสอบได้ทันที
2. ให้ intent ที่ชัดเจนตอบแบบ fast-path ก่อน LLM
3. ให้คำถามทั่วไปเข้า LLM จริง แต่ต้อง warm-up และมี fallback ที่ไม่หลอกผู้ใช้
4. เสียงต้องใช้ไฟล์เดิมแบบ overwrite เพื่อลดการสะสมไฟล์
5. Dashboard ต้องเห็นสถานะจริงของ AI, ESP32, sensor, voice และ external APIs

## Step 1: AI Warmup และสถานะ LLM

สถานะ: เสร็จแล้ว

งานที่ต้องมี:
- เพิ่ม endpoint สำหรับปลุก LLM ด้วยตัวเองก่อนเดโม
- เพิ่ม endpoint สำหรับพัก LLM เมื่อต้องการประหยัดเครื่อง
- เปิด keep-awake loop ใน demo mode เพื่อไม่ให้โมเดลหลับเองระหว่างนำเสนอ
- แสดงสถานะ LLM บน dashboard
- บอกได้ว่าโมเดลพร้อมไหม, warm แล้วไหม, ใช้โมเดลอะไร, latency ล่าสุดเท่าไร
- ถ้า Ollama หรือโมเดลไม่พร้อม ต้องแสดง degraded แบบอ่านง่าย

เหตุผล:
- ปัญหาหลักของ local LLM คือช้ารอบแรก
- ถ้าปลุกโมเดลไว้ก่อน ผู้ใช้จะรู้สึกว่ากำลังคุยกับ AI จริง ไม่ใช่ bot ที่รอ timeout

## Step 2: Persona น้องฟ้า

สถานะ: รอทำถัดไป

งานที่ต้องมี:
- ปรับ system prompt ให้มีตัวตนชัดเจนชื่อ "น้องฟ้า"
- ตอบสั้น เป็นธรรมชาติ และไม่แข็ง
- รู้ตัวว่าเป็นผู้ช่วยบ้านอัจฉริยะ แต่ยังคุยเรื่องทั่วไปได้
- ไม่อ้างว่าทำสิ่งที่ระบบยังไม่ได้ทำจริง

## Step 3: General Chat Quality

สถานะ: รอทำ

งานที่ต้องมี:
- ปรับ fallback ให้แยกกรณี timeout, model down, และคำถามทั่วไป
- ถ้า LLM ช้ารอบแรก ให้ UI บอกว่า "กำลังคิด" แทนการดูเหมือนระบบพัง
- ใช้ streaming สำหรับข้อความทั่วไปต่อไป
- จำกัดคำตอบให้เหมาะกับการสร้างเสียง

## Step 4: Voice Conversation Polish

สถานะ: รอทำ

งานที่ต้องมี:
- ลดโอกาส speech recognition เก็บเสียง AI กลับมาเป็นคำถาม
- ปรับ cooldown หลังเล่นเสียง
- ให้ wake word กลับไปรอฟังเองหลังจบบทสนทนา
- แยกสถานะ "รอคำปลุก", "กำลังคุย", "กำลังคิด", "กำลังพูด" ให้ชัด

## Step 5: LINE News Workflow

สถานะ: รอทำ

งานที่ต้องมี:
- เมื่อถามข่าว ให้ตอบ headline สั้น ๆ
- ถ้าขออ่านรายละเอียด ให้เสนอส่งลิงก์เข้า LINE ก่อนอ่านยาว
- รองรับหัวข้อข่าว เช่น การเงิน, การเมืองไทย, AI, เทคโนโลยี
- ไม่อ่านข่าวยาวด้วยเสียงโดยไม่จำเป็น

## Step 6: Hardware Readiness

สถานะ: รอทำ

งานที่ต้องมี:
- ตรวจ ESP32 online/offline ให้ตรงกับ heartbeat จริง
- DHT22 และ PIR ต้องแสดงสถานะสดใน dashboard
- Relay command queue ต้องตรวจง่ายก่อนเดโม
- INMP441 ให้เริ่มจาก sound activity/wake signal ก่อน ยังไม่ใช้เป็น STT หลัก

## Step 7: Presentation Script

สถานะ: รอทำ

งานที่ต้องมี:
- สคริปต์เปิดเครื่องหลัง reboot
- สคริปต์ demo 5-7 ฉาก
- รายการคำถามสำรอง
- วิธี fallback ถ้า internet, API, หรือ hardware ไม่พร้อม

## ลำดับแนะนำถัดไป

1. ทำ Step 1 ให้เสร็จและ commit
2. ทำ Step 2 ปรับ persona น้องฟ้า
3. ทำ Step 3 ปรับคำตอบทั่วไปให้ฉลาดขึ้น
4. ทำ Step 4 ปรับ wake word/voice UX
5. ทำ Step 5 ส่งข่าวเข้า LINE ให้ครบ flow
